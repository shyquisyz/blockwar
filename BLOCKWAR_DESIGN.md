# BLOCK WAR — 게임 설계 문서

> 최종 업데이트: 2026-06-12

---

## 핵심 재미 정의

> **"적을 추적한 후 원샷 또는 짧은 교전으로 처치하는 순간"**

모든 시스템은 이 감정을 강화하는 방향으로 설계한다.
**Find → Listen → Predict → Shoot First**

---

## 개발 우선순위

```
✅ 완료    🔧 진행중    📐 설계완료    ⏸ 보류
```

| 순위 | 모드 | 상태 | 목표 |
|---|---|---|---|
| 1 | 온라인 1 vs 1 | 🔧 진행중 | 친구 1명과 인터넷 실전 플레이 |
| 2 | 협동 모드 (2P vs 4AI) | 📐 설계완료 | 친구 1명만 있으면 언제든 |
| 3 | AI 대전 개선 | 📐 설계완료 | 단독 플레이 완성도 향상 |
| 4 | 2 vs 2 | ⏸ 보류 | 4명 필요 |
| 5 | 3 vs 3 | ⏸ 보류 | 6명 필요 |
| — | 스토리 모드 | ⏸ 보류 | 콘텐츠 규모 문제 |

**핵심 목표**: 혼자서도 재밌고, 친구 1명만 있으면 더 재밌는 게임

## 1순위 — 온라인 1 vs 1

### 현재 구현 상태

| 기능 | 상태 |
|---|---|
| 방 생성 / 방 참가 (5자리 코드) | ✅ 구현 |
| WebSocket 릴레이 서버 | ✅ 구현 |
| 플레이어 상태 동기화 (20Hz) | ✅ 구현 |
| 총격 시각화 동기화 (트레이서/사운드) | ✅ 구현 |
| 피격 이벤트 전송 | ✅ 구현 |
| 사망 동기화 | ✅ 구현 |
| 재경기 (라운드 종료 후 자동 재시작) | ✅ 구현 |
| 서버 배포 | 🔧 진행중 |

### 아키텍처: 대칭 P2P

```
[플레이어A (팀0, 왼쪽)] ←── 릴레이 서버 ──→ [플레이어B (팀1, 오른쪽)]
       ↓ 매 50ms 전송                              ↓ 매 50ms 전송
  {type:'state', x,y,angle,hp,alive,wk,reload}
       ↓ 즉시 전송 (hit/shot)
  {type:'hit', dmg, head}
  {type:'shot', wk, ox,oy, ex,ey}
```

- 각 클라이언트가 자신의 플레이어에 대해 권위(Authority)를 가짐
- 내 총이 상대에게 맞으면 hit 이벤트 전송 → 상대방이 자신의 hp 차감
- 시야/안개는 각 클라이언트가 독립적으로 계산
- 서버는 메시지 중계만 담당 (게임 로직 없음)

### 릴레이 서버

- **위치**: `server/server.js`
- **배포**: Render.com 무료 티어
- **URL 설정**: `blockwar.html` → `const SERVER_URL`
- **프로토콜**: WebSocket, JSON

## 2순위 — 협동 모드 (2P vs 4 AI)

### 목표

- 플레이어 2명이 같은 팀 (팀0)
- 적 봇 4명 (팀1)
- 라운드제: 적 전멸 시 승리
- 방 코드로 온라인 연결

### 아키텍처: 호스트 권위 (Host-Authority)

1v1과 달리, AI 시뮬레이션을 두 클라이언트가 독립적으로 실행하면 동기화가 불가능하다.
**호스트 한 명이 전체 시뮬레이션(플레이어 2명 + 봇 4명)을 실행하고 상태를 전송한다.**

```
[호스트 (플레이어0)]                    [클라이언트 (플레이어1)]
 ├─ 자신의 입력 처리                      ├─ 매 프레임 입력 전송 →→→
 ├─ 수신한 클라이언트 입력으로              │  {type:'input', keys, mouseX, mouseY,
 │   플레이어1 엔티티 조종                  │   mouseL, mouseR, seq}
 ├─ 봇 4마리 AI 실행                      │
 ├─ 전투 판정 (히트스캔)                   ├─ 전체 게임 상태 수신 ←←←
 └─ 매 50ms 전체 상태 전송 →→→           └─ 상태 기반 렌더링
    {type:'gamestate',                        + 자신의 이동은 로컬 예측
     players:[p0,p1],
     bots:[b0,b1,b2,b3],
     drops:[...],
     events:[shots,deaths,bloods]}
```

### 호스트 → 클라이언트 상태 패킷 (50ms마다)

```json
{
  "type": "gamestate",
  "seq": 1234,
  "players": [
    { "id":0, "x":300, "y":400, "angle":1.5, "hp":80, "alive":true, "wk":"ak", "reload":false },
    { "id":1, "x":320, "y":380, "angle":1.2, "hp":100, "alive":true, "wk":"smg", "reload":false }
  ],
  "bots": [
    { "id":2, "x":900, "y":300, "angle":0, "hp":100, "alive":true, "wk":"ak", "state":"PATROL" },
    { "id":3, "x":1200, "y":600, "angle":3.1, "hp":60, "alive":true, "wk":"ak", "state":"ENGAGE" }
  ],
  "drops": [{ "x":800, "y":500, "weaponKey":"ak", "mag":15, "reserve":30 }],
  "events": [
    { "t":"shot", "wk":"ak", "ox":300, "oy":400, "ex":850, "ey":360 },
    { "t":"death", "id":3, "x":1200, "y":600 },
    { "t":"blood", "x":850, "y":360 }
  ]
}
```

### 클라이언트 → 호스트 입력 패킷 (매 프레임)

```json
{
  "type": "input",
  "seq": 5678,
  "keys": { "W":true, "A":false, "S":false, "D":true, "Space":false, "Shift":false },
  "mouseX": 450, "mouseY": 300,
  "mouseL": true, "mouseR": false
}
```

### 구현 변경사항 (1v1 대비)

| 항목 | 1v1 | 협동 |
|---|---|---|
| 팀 구성 | 0팀 vs 1팀 | 0팀(2명) vs 1팀(봇4) |
| 권위 | 각자 자신 | 호스트 전체 |
| 봇 실행 | 없음 | 호스트만 실행 |
| 상태 전송 | 양방향 대칭 | 호스트→클 (full state) |
| 전투 판정 | 발사자 로컬 | 호스트 |
| 클라이언트 예측 | 필요없음 | 자신의 이동만 예측 |

### 릴레이 서버 변경

서버는 게임 타입을 알 필요 없음. 방 생성 시 타입만 기록:
```json
{ "type": "create", "mode": "coop" }
```
두 클라이언트가 연결되면 start 메시지에 게임 타입 포함:
```json
{ "type": "start", "yourSlot": 0, "mode": "coop" }
```

## 3순위 — AI 대전 개선

### 미구현 항목

| 기능 | 내용 |
|---|---|
| 총알 궤적 0.8초 유지 | 트레이서 life 0.8~1.0초 (이미 구현됨) |
| 발소리 거리 제한 | step 링 230px 이내만 표시 (이미 구현됨) |
| 피 흔적 15~20초 유지 | bloodStain life 15-20s (이미 구현됨) |
| 시체 정보 표시 | 8초 맥박링, 5초 "!" (이미 구현됨) |
| 무기 줍기 (F키) | F키 수동 픽업 (이미 구현됨) |
| 거리별 정확도 | 원거리일수록 조준 오차 증가 (이미 구현됨) |
| **기습 보너스** | PATROL/INVESTIGATE 상태 적 → 1.5× 피해 (이미 구현됨) |

대부분 이미 구현됨. 추가 개선 여지:
- 봇 무기 다양화 (SMG/AWP 봇 추가)
- 봇 수류탄 / 섬광 (장기 로드맵)
- 봇 팀 전술 (pincer 이동)
